## 一、背景与参考

**参考站点（英文）**: [https://ccunpacked.dev/](https://ccunpacked.dev/)（*Claude Code Unpacked*）

该站特点：**单页长卷 + 编号区块**（01 Agent Loop、02 架构树、03 工具系统、04 斜杠命令、05 隐藏/实验特性）+ **从源码映射的策展**；交互与视觉完成度高，但语言为英文，且与 **本仓库课程结构**、**`ccsource` 版本** 未绑定。

**本计划目标**：在 **遵守版权与署名** 前提下，将 **同类信息架构（IA）** 以 **中文** 落地到本站 `site/`，并 **链回现有讲义与专题**，形成长期可维护的 **「解构 / 导览」入口** —— **不替代** S/D 正文，**不**整站机翻第三方长文。

## 三、信息架构（五区块 — 与参考站对齐）

以下为 **MVP 必备区块**；标题可中文化，侧栏锚点建议用 `id` 与下表一致或映射。

| 区块 | 中文建议标题 | 内容要点 | 本站出口（示例） |
|------|----------------|----------|------------------|
| **01** | 智能体主循环 | 从输入到 `tool_result` 回流、退出条件；**不**粘贴大段英文 | `s01.html`、`d01.html` |
| **02** | 架构导览 | `utils`/`commands`/`tools`/`services` 等 **目录级心智模型**；数字可与首页或脚本统计一致 | `topic-sourcemap.html`、`handbook.html` |

## 四、分阶段路线图

### 阶段 A — MVP（本期 v2.5.2 收口目标）

- 新增 **`site/<stem>.html`**（`stem` 由 **Agent 1** 敲定，建议 `topic-cc-unpacked-zh`）。
- 五区块 **全中文**；统计数字 **口径与首页或实测一致**。
- **主导航**（及侧栏若需要）增加入口。
- **`site/md/<stem>.md`** + 页脚 **GitHub 互链**；**`check_site_md_parity.py` 通过**。

### 阶段 B — 数据化（可选，同计划内由 Agent 2 启动）

- `data/cc-overview.json`（或约定路径）描述分组与链接。
- `tools/gen_cc_overview.py`（或等价）生成片段或文档，**减少手工改 HTML**。

### 阶段 C — 交互增强（远期 / 可另开 v2.6.x）

- Agent Loop **步进视图**（纯 CSS + 少量 JS 或小 bundle）；**渐进增强**（无 JS 可读全文）。
- 与 **发版监督** 联动：每次大版本更新统计与「新增命令/工具」diff。

## 五、技术约束

- **不引入** React/Next 等新构建链（除非全站另立子项目）。
- 样式复用 **`site/css/style.css`**；暗色/主题与现有一致。
- 新资源图片若需：**`site/images/`** 下独立子目录，控制体积。

---

## 六、风险与合规

- **勿复制** 参考站的 **长段英文说明、定制插图、私有脚本**；**工具名、命令名** 属事实信息，**解释性段落须自写**。
- **数字漂移**：工具/命令数量随版本变；依赖 **版本锚定 + 定期人工/脚本更新**。
