## 一、目标

1. **仓库内**为 `site/` 下每个独立对外页面（`.html`）维护一份对应的 **Markdown 源文件**，统一放在 **`site/md/`** 目录，并纳入 Git 版本管理。
2. **HTML 页面**上增加稳定可见的入口：**链接到 GitHub 上该页的 `blob/main/site/md/<同名>.md`**（新窗口打开），便于 PR、审阅与纯文本阅读。
3. **MD 与 HTML 关系**（执行约定）：
   - **v2.5.0 第一期**：以「**结构化占位 + 元信息 + 指向在线 HTML**」为主；正文可写摘要或章节提纲，**不要求**首版即与 HTML 全文逐字等价（避免单次 PR 体量爆炸）。
   - **后续迭代**：可按讲次逐步把 MD 扩写为与 HTML 对齐的正文，或引入半自动生成流程（另开计划）。

---

## 二、路径与链接规范

| 类型 | 路径 |
|------|------|
| HTML | `site/<stem>.html` |
| Markdown | `site/md/<stem>.md` |
| GitHub 原文链接（固定前缀） | `https://github.com/Harzva/learn-likecc/blob/main/site/md/<stem>.md` |
| 在线渲染页（可选写在 MD 里） | `https://harzva.github.io/learn-likecc/<stem>.html` |

- **例外**: `index-old.html` 若仍保留：对应 `site/md/index-old.md`；若计划废弃该页，可在 MD 中注明 deprecated 并仅保留链接说明。

### HTML 侧推荐 UI（由各 Agent 统一实现）

在**页脚前**或**主导航栏辅助区**增加一行文案 + 链接，例如：

```html
<p class="md-source-link"><a href="https://github.com/Harzva/learn-likecc/blob/main/site/md/s01.md" target="_blank" rel="noopener noreferrer">📝 本站页的 Markdown 源文件（GitHub）</a></p>
```

样式可复用现有 `.footer` / `.section-desc`，必要时在 `site/css/style.css` 增加 `.md-source-link`（字号略小、与版权行区分）。

### MD 文件推荐模板（每个 `<stem>.md` 顶部）

```markdown
# <与 HTML title 一致的标题>

> **在线页面**: https://harzva.github.io/learn-likecc/<stem>.html  
> **本文件**: `site/md/<stem>.md`  
> **说明**: 正文可与 HTML 逐步对齐；当前版本以导航与概要为主。

## 概要

（2～5 句或小节提纲）

## 正文（可选 / 待补全）

（后续迭代填充）
```
