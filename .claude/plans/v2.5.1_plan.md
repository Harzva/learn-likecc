## 一、目标

1. **CI**：在 GitHub Actions 中对变更 `site/**` 的 PR / `main` 推送自动运行 `check_site_md_parity.py`，防止回归。
2. **MD 正文深化**：在保持 HTML 为**主要阅读面**的前提下，将各 `site/md/<stem>.md` 从「仅概要」提升为**可读讲义草稿**——至少包含与 HTML 主要章节对应的**目录（h2/h3）+ 每节简短摘要**（可手工整理，也可半自动从 HTML 抽取标题后人工润色）。
3. **编辑约定**：在 `site/md/README.md` 写清「何时改 MD、何时改 HTML、谁为权威」以避免双份漂移（本期以 **HTML 权威、MD 跟进** 为默认）。
