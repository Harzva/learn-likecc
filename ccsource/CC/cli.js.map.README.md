# cli.js.map - Claude Code Source Map 泄露事件

## 什么是 Source Map？

Source Map 是一张**对照表**。

| 阶段 | 状态 | 说明 |
|------|------|------|
| 开发时 | 源码 | 变量名清楚，结构规整，人能看懂 |
| 发布时 | 压缩/编译 | 体积更小，运行方便，人难看懂 |
| Source Map | 对照表 | 告诉调试器压缩代码与源码的对应关系 |

Source Map 本来只是调试用的辅助材料。问题在于，一旦被**误打进正式发布包**，它就变成了**源码导航图**。

## 安全架构分析

Claude Code 的安全设计**比预期更重**：

```
┌─────────────────────────────────────────────────┐
│                   用户输入                        │
└─────────────────────┬───────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│              多层权限验证                         │
│  • 工具调用前审核                                 │
│  • 上下文风险评估                                 │
│  • 用户确认机制                                   │
└─────────────────────┬───────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│              高风险操作单独审核                    │
│  • Shell 执行                                    │
│  • 文件写入                                       │
│  • 敏感命令                                       │
└─────────────────────┬───────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────┐
│              沙箱环境执行                         │
│  • 外部命令隔离                                   │
│  • 插件隔离                                       │
└─────────────────────────────────────────────────┘
```

### 安全要点

- **不是**"模型想执行什么就直接执行"
- Shell、文件写入有**单独审核路径**
- 外部命令和插件尽量扔进**沙箱**
- 根据上下文、风险级别、用户确认做决策

## 代码质量

### 架构评价

| 方面 | 评价 |
|------|------|
| 整体架构 | ✅ 扎实 |
| 权限设计 | ✅ 优秀 |
| 代码风格 | ⚠️ 有技术债 |

### 典型问题

- 某些文件特别长 (如 `src/cli/print.ts`)
- 某些函数层层嵌套
- 有些地方"先跑起来再说"

### 有趣细节

识别用户负面情绪用的是**正则匹配关键词**，不是花哨的情感分析模型。

> 能用一个简单办法解决的事，团队未必愿意专门上个更贵更复杂的方案。

## 重要边界

> 泄露的是**应用层代码**，不是**模型权重**。

外界拿到的是：
- ✅ Claude Code 产品外壳和控制层代码
- ❌ Claude 模型本体

但 AI 编程产品最有看头的部分（权限、工具调用、上下文压缩、记忆落地）都在这里了。
