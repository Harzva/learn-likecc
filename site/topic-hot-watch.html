热门话题专项
先聚合，再路由，再沉淀成专题

这页不是另一个新闻站，而是 Learn LikeCode 的热点 intake 面：先汇总外部热点源，再把值得讲的内容分流到 Agent、RAG、AI 杂货铺、工具链或大模型专题里。当前第一批受控来源以 AIbase AI日报、RSS 友好英文博客 和 官方产品 / 生态博客 RSS 为起点。

看当前摘要 看维护规则

这条专项解决什么问题

站内需要一个“先收热点，再决定写到哪”的 intake 层。 否则每次看到 AI 日报、外部快讯、产品榜单和 RSS 更新，都只能靠人工临时记忆，很难形成长期维护的站内路线。

  • 它负责把公开热点源聚合成一份轻量快照。
  • 它不做全文镜像，只保留标题、链接、日期和一行判断。
  • 它最终要把线索路由回站内现有专题，而不是把所有内容都困在一页里。

第一批来源池

AIbase AI日报

代表“公开日更聚合页”这类来源。它很适合做热点 intake，但需要二次路由，不能直接当最终专题。

Daily Dose of Data Science

代表 RSS 友好源。稳定、结构清晰，适合半自动抓取和后续送往 Agent / RAG 热点页。

Hugging Face Blog

代表官方产品 / 生态博客 RSS。适合补模型发布、embedding / reranker、infra 与开源生态向热点，再路由到工具链 / 大模型 / RAG 相关页。

当前来源池按三条 lane 管理：公开聚合页负责 discovery，独立 RSS 负责稳定工程评论，官方博客 RSS 负责 primary-source 校准。这样做的目的，是避免某一类源把整个 intake 面压扁成单一视角。

技能入口：.claude/skills/hot-topic-curation/SKILL.md；源配置：site/data/hot-topic-sources.json。

抓完之后写到哪里

Agent 技术热点

给 Agent、CLI、workflow、managed agent、coding shell 这一类新闻做摘要和短评。

 RAG 技术热点

给 retrieval、grounding、eval、向量库和 indexing 这一类内容做分流。

 AI杂货铺

当热点更像“入口发现”或“工具地图”时，先回到杂货铺，而不是硬写成长文。

 工具链 / 大模型专题

当热点直指 API、provider、价格、模型版本或 infra 变化时，适合送往工具链或大模型主线。

维护规则

  1. 先改 site/data/hot-topic-sources.json，不要把新源只写进文案里。
  2. 运行 python3 tools/fetch_hot_topic_sources.py --write 刷新本地快照。
  3. 新增源后，先判断它属于 discovery digest、independent RSS 还是 official blog RSS，再决定默认 route hints，不要把所有 RSS 当成同一类。
  4. 如果来源池或路由逻辑有明显变化，同步更新这页和对应热点页。
  5. 如果某条热点足够重要，再交给 keyword-site-topic 做深一层的站内稿件。
