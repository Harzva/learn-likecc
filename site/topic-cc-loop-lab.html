Decision Guide

先按问题选入口

生产级教程最重要的不是把内容堆满，而是让学习者在 10 秒内知道自己该点哪里。

我还不理解循环

从 Agent Loop 动态模拟器开始

看一次完整 turn 如何走过模型、工具、观察和最终回答，先建立运行直觉。

 我想练命令和配置

进入脚本启示仿真器

把命令脚本、终端练习、配置生成和测验串成一条可重复训练路径。

 我想拆请求体

进入 Trace Prompt 仿真器

 我想接真实事件

看本地 SSE 联调

启动 relay，把演示事件流接到浏览器里，理解仿真与真实运行时之间的接口。

Simulation Paths

三种在线仿真器

每个入口都对应一种认知负载：先看循环，再练脚本，最后拆 prompt 请求。

01 Agent Loop

动态模拟器

12 步回放一次完整 Agent 循环，适合第一次建立“模型为什么会调用工具、工具结果如何回灌”的直觉。

输出：运行链路心智图 进入模拟器 → 02 Script Insight

脚本启示仿真器

把命令脚本、终端练习、配置生成和测验放进一个交互实验室，适合课程训练和 workshop。

输出：可练习的命令套路 进入练习室 → 03 Trace Prompt

Prompt 请求仿真器

结合三份 cc-route trace，逐层拆解 system、tools、runtime reminder 与用户输入如何组装成请求。

输出：prompt 结构阅读能力 进入拆解台 →

Learning Runway

建议按这条路线学

这个顺序能让概念、操作和源码之间互相托住，不会一下子掉进细节里。

Step 1看循环

用 Agent Loop 模拟器跑一遍完整 turn。

Step 2做练习

用 Script Insight 把命令、配置、测验跑起来。

Step 3拆请求

用 Trace Prompt 读懂真实请求里的层级。

Step 4接事件

用 SSE 联调理解仿真和真实运行时的接口。
