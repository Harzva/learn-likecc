Contract

这块控制面的默认规则

  • 长循环任务优先来自 Task N + dedicated plan file 这套结构，而不是继续把所有细节塞在 prompt.md 里。
  • 一个任务如果没完成，就优先续跑，而不是被新任务轻易挤掉。
  • 任务驾驶舱只负责把状态读出来，不替代真正的执行日志；更细的细节依然应该落回 plan、evolution 和 repo 变更里。
  • 控制面默认给出 GitHub Pages 实时页面、GitHub blob 证据面，以及按计划语义分组后的任务视图。
  • 新增长循环任务后，记得重新生成这页数据：python3 tools/build_loop_task_board.py。
