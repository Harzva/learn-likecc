## 一句话

`QueryEngine` 持有会话历史与成本统计；**真正的 tool-use 循环**在 `run_query()`：每轮先可选压缩，再流式调模型，若有 `tool_uses` 则执行工具并把 `ToolResultBlock` 拼回消息，直到模型不再要工具或触达 `max_turns`。

## 与 S01 的映射

| S01 概念 | OpenHarness 落点 |
|----------|------------------|
| QueryEngine 配置袋 | `QueryEngine` 构造参数 + 运行时的 `QueryContext` |
| while 直到非 tool_use | `run_query` 中 `while` + `if not final_message.tool_uses: return` |
| 消息累积 | `messages: list[ConversationMessage]` 原位追加 assistant / user(tool results) |
| 流式 UI | `AsyncIterator[StreamEvent]`（`AssistantTextDelta` 等），而非只返回最终块 |

## 特有点（相对课内 TS 伪代码）

- **`max_turns`**：单次用户输入下 Agent 轮数上限（默认 `QueryEngine` 传 `8`，`QueryContext` 默认上限更高，以构造为准）。
- **每轮前 `auto_compact_if_needed`**：在调用模型前做上下文压缩（microcompact / LLM 摘要），S01 伪代码未展开。
- **多工具 `asyncio.gather`**：多个 tool_use 时并发执行，再统一发完成事件。
- **`continue_pending`**：无新用户句也可续跑（工具结果已就绪时）。
