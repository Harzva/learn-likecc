## 概要

本讲介绍 Claude Code 中 **Agent 主循环**：模型响应、工具调用决策、执行结果回注与多轮对话状态。涵盖 `stop_reason`、消息累积与典型故障场景的处理思路。

## 目录（对照 HTML）

章节标题与 [`site/s01.html`](../s01.html) 内 `h2` / `h3` 对应。**Mermaid 导图、完整 TypeScript/Python 双栏代码与练习题干以在线页为准。**

### 🎯 问题

模型无法直接触碰文件系统与进程，若没有自动循环，每次工具结果都要人工贴回上下文。本节点出「人肉当循环」的痛点。

### 💡 解决方案

用「用户 → LLM → 工具执行 → tool_result 回流」的闭环替代手动拼接；循环持续到 `stop_reason` 不再表示需要工具为止。在线页含 ASCII 示意框图。

### 🔍 源码分析

#### QueryEngine.ts 核心结构

概括 `QueryEngineConfig` 的职责：工作目录、工具集、斜杠命令、MCP、子 Agent 定义、权限回调与 AppState 读写等装配关系。具体字段表与代码见在线页。

#### Agent Loop 伪代码

用伪代码逐步展示：调用 LLM、追加 assistant、根据 `stop_reason` 判断是否退出、执行 `tool_use` 并构造 `tool_result`、再写回 messages。可与 D01 深挖的状态机对照阅读。

### 🔑 关键设计点

围绕消息累积、工具并行与错误恢复等拆成若干小卡片式论点；在线页用短代码片段强化记忆。

### 📐 设计模式

把主循环与周边模块的关系上升到可复用的结构模式（如职责划分、扩展点），便于迁移到其他 Agent 宿主实现。

动手题引导你在最小环境里复现「一轮 tool 调用 + 回流」，巩固循环语义。
