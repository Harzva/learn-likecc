## 各节摘要（对照 HTML）

### 导语

从 Karpathy 的 LLM Wiki 讲起：真正重要的不是临时把资料检索出来，而是让模型持续维护一套会增长、会修订、会交叉引用的知识库。

### 为什么值得做

解释 RAG 式临时问答为什么“不沉淀”，以及为什么要把重点转向“摄入时编译知识”。知识在进入 wiki 的那一刻才真正开始复利增长。

### 三层结构：GitHub、Wiki、Agent 各管一层

把这条线拆成三层：

- `GitHub / Raw Sources` 管底层原始资产与版本
- `Wiki / Obsidian` 管阅读、检索、链接和展示
- `LLM / Agent` 管编译、更新、归档和交叉引用

### 更像工程而不是笔记的工作流

这一节提炼“摄入 / 查询 / 检查”三步工作流，强调个人知识库更像工程维护，而不是一次性记笔记。

### 和站内哪些线互补

把这页与 `topic-memory-harness.html`、`topic-websites.html`、`tutorial.html#skills` 连接起来：Memory 偏机制，站点推荐偏入口，Skills 偏规则化与工作流固化。

### 一点现在的感想

放入你确认过的那段收尾感想，包括：

- GitHub / Wiki / Agent 的三层分工
- 知识从笔记走向能力包
- `everything can be skill`
- “这何尝不是一种赛博永生”
