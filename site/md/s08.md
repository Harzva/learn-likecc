## 概要

本讲涉及 **任务管理**：多步骤目标的拆解、进度跟踪与中断恢复，以及与 Agent 长对话的配合方式。

## 目录（对照 HTML）

### 🎯 问题

长对话里「待办、进行中、阻塞」若只靠自然语言，容易丢失依赖关系与验收标准，也难以并行协作。

### 💡 解决方案

引入显式 Task 实体：标题、描述、状态与关联上下文，使模型与用户能共同维护一张可机器解析的进度表。

### 🔑 Task 数据结构

字段级说明（标识、状态、父子关系等）如何支撑列表视图与后续自动化。类型/接口代码见在线页。

### 📊 状态流转

描述从创建、执行、完成到失败/取消的典型迁移；在线页可用表格或示意表达合法边。

### 🔧 Task 工具

列举创建、更新、查询任务等面向模型的 tool 名称与语义，及其在 Agent Loop 中的调用时机。
