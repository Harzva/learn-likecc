## 一句话

每个工具是 **`BaseTool` 子类**：`name` / `description` / **`input_model`（Pydantic）** 生成 API schema；**`ToolRegistry`** 按名注册；`run_query` 里 **`model_validate` → 权限（见 S03/OH03）→ `execute(context)` → `ToolResult`**。

## 与 S02 的映射

| S02 概念 | OpenHarness |
|----------|-------------|
| `inputSchema`（zod） | `input_model: type[BaseModel]` + `model_json_schema()` |
| `execute(params, context)` | `async execute(arguments: BaseModel, context: ToolExecutionContext)` |
| `hasPermission` | 不在 BaseTool 上；统一在 `query._execute_tool_call` 里用 `PermissionChecker` + `is_read_only()` |
| 工具注册表 | `ToolRegistry.register` / `create_default_tool_registry()` |

## 串联 OH01

`query.py` 中：`tool = context.tool_registry.get(tool_name)` → `tool.input_model.model_validate(tool_input)` → `permission_checker.evaluate(..., is_read_only=tool.is_read_only(parsed_input))` → `tool.execute(parsed_input, ToolExecutionContext(...))`。

## 建议阅读顺序

1. `base.py`：`BaseTool`、`ToolRegistry`、`ToolResult`、`ToolExecutionContext`。  
2. `file_read_tool.py`：只读工具与 `is_read_only`。  
3. `bash_tool.py`：可写副作用、超时与输出截断。  
4. `__init__.py`：`create_default_tool_registry` 与 MCP 工具挂载方式。
