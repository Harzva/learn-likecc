## 各节摘要

### 子专题入口

把现有结构导览页降成子专题，同时新增 `Superset / Hermes / Cabinet / Multica` 四条解构页，让“拆结构”“拆调度”“拆长期 agent 工作台”“拆 managed agents 平台”放到同一专题下。

### 为什么要把它升成专题

现在要拆的已经不只是 Claude Code 自己，还包括 Superset 这类外层 orchestration shell，以及 Cabinet / Multica 这类更接近 managed agents 的工作台与平台。所以“庖丁解牛”应该变成方法论，而不是只绑定一个页面。

### 推荐阅读顺序

建议：

1. 先看 `Claude Code 解构`
2. 再看 `Superset 解构`
3. 然后看 `Cabinet / Multica` 对比 knowledge-first 与 runtime-first 两条 managed agents 路线
4. 最后回到 Like Code 判断哪些能力要内生化
