## 概要

本讲介绍 **子 Agent 分叉**：何时拆分子任务、与子进程/并行调用的关系，以及结果合并回主会话的方式。

## 目录（对照 HTML）

章节标题与 [`site/s06.html`](../s06.html) 内 `h2` 对应。**内置 Agent 类型表、YAML 式定义示例与隔离说明的长代码见在线页。**

### 🎯 问题

单一主循环难以同时处理「探索性检索」与「需隔离上下文的实现」；需要受控的子会话以免污染主对话与权限边界。

### 💡 解决方案

通过预置或自定义 Agent 定义，在独立上下文/工具集中运行子任务，再把结构化结果合并回父会话。

### 🔑 内置 Agent 类型

列举框架提供的典型子代理角色及其适用场景（如专项调查、测试运行等），具体名称与描述见在线页卡片/列表。

### 📐 Agent 定义

说明声明式字段：名称、系统提示、工具白名单、模型偏好等如何约束子循环行为。配置样例见在线页。

### 🔒 隔离模式

讨论文件系统、环境变量、网络与密钥可见性等隔离维度，以及合并时冲突处理的高层原则。
