## 概要

本讲说明 **工具系统**：内置与自定义工具的注册方式、调用载荷与返回约定，以及工具失败时的重试与降级策略在 Agent 循环中的位置。

### 🎯 问题

仅有语言模型不足以完成读写、搜索与执行命令；需要类型化、可审计的工具层把「意图」变成宿主可执行动作。

### 💡 解决方案

在统一抽象下注册工具：入参 schema、异步执行函数、可选的 UI 渲染钩子，使模型输出 `tool_use` 时能落到一致实现。

### 🔍 源码分析

#### Tool.ts 核心结构

从类型层说明 `Tool`/`ToolResult` 一类契约：执行签名、上下文注入与结果形状。完整泛型定义见在线页。

### 🔑 内置工具

用卡片形式罗列 Read、Edit、Write、Bash、Grep、WebSearch 等主线能力及一句职责说明，对应 S01 循环里最常见的 tool 名称。
