# OH03: Permission Model（OpenHarness · 对标 S03）

> **在线页面**: https://harzva.github.io/learn-likecc/oh03.html  
> **本文件**: `site/md/oh03.md`  
> **对照**: [S03 Permission Model](s03.md)  
> **上游路径**: `permissions/checker.py`、`permissions/modes.py`、`config/settings.py`（`PermissionSettings`）、`engine/query.py`（判定与确认）、`ui/permission_dialog.py`（TUI 提示）

## 一句话

**`PermissionChecker.evaluate`** 根据 `PermissionSettings`（mode、白/黑名单、路径 glob、命令 deny 模式）和 **`is_read_only`** 得到 **`PermissionDecision`**；若「未允许但需确认」且注入了 **`permission_prompt`**，则由 UI 询问后再决定是否继续执行工具。

## 与 S03 的映射

| S03 概念 | OpenHarness |
|----------|-------------|
| Auto-allow / Ask / Deny | `allowed` + `requires_confirmation`；或显式 deny reason |
| PermissionMode | `PermissionMode`：`DEFAULT` / `PLAN` / `FULL_AUTO`（比课内 TS 枚举更窄） |
| 路径/命令策略 | `path_rules`（glob）、`denied_commands`（glob） |
| 用户确认 | `QueryContext.permission_prompt`；默认 TUI 为 `ask_permission` |

## 串联 OH02

工具层提供 **`is_read_only`**；`query._execute_tool_call` 在 **`model_validate` 之后** 解析 `file_path` / `command` 再调用 `evaluate`。

## 建议阅读顺序

1. `modes.py` → `PermissionSettings`  
2. `checker.py`：`evaluate` 分支顺序（deny list → allow list → path → command → FULL_AUTO → read-only → PLAN → default 需确认）  
3. `query.py` 权限分支与 `permission_dialog.py`
