## 各节摘要（对照 HTML）

### 导语

解释这页为什么存在：不是收资讯，而是收“隔一阵子还会回来打开”的长期入口。现在把 `zread.ai` 放在“repo 阅读入口”这一格，把 `karpathy.ai` 放在“长期知识主页”这一格，两类价值并列呈现。

### Zread.ai：为什么该放进这页

这节说明 `zread.ai` 值得推荐的原因：它的首页直接给出 `Discover Repos`、`Trending Repos`、`Your Private Repos`、`Zread MCP Server`、`Zread CLI for Local Repos` 等入口，明显不只是单纯外链页，而是在把 repo 做成可读、可问、可接入工作流的对象。

这一节同时补了一张和 `projects/rep2course` 的差异表。结论很明确：`zread.ai` 更像 repo 阅读与发现产品，`rep2course` 更像 repo 到课程资产的生产流水线；一个偏前置侦察，一个偏后置沉淀，两者是互补关系。

### Karpathy 的 LLM Wiki 指南，为什么值得单独记一下

基于你提供的中文整理，把这篇文章压成站内导读：它的核心不是再造一个新名词，而是说明“知识为什么总留不下来”，以及为何应当从临时检索的 RAG 心智转向持续维护的 Wiki 心智。文中还提炼了三层结构：`Raw Sources`、`Wiki`、`Schema`，以及“摄入 / 查询 / 检查”的长期维护工作流。

### 教程入口

包含 Karpathy 的 LLM 长视频、神经网络训练经验文、`CS231n`，以及 Sebastian Raschka 的 `LLM Architecture Gallery`。这一组更偏“补理解”和“补直觉”。

### 个人主页

收两个风格稳定、长期可回看的个人主页：`karpathy.ai` 与 Sebastian Raschka 主页。重点不是站点设计，而是它们如何把长期资产组织成低摩擦入口。

### 值得反复回看的 GitHub 仓库

以 `micrograd`、`nanoGPT`、`char-rnn`、`arxiv-sanity-preserver` 为主，分别对应自动求导、最小 GPT 实现、序列建模历史脉络和研究检索工具。
