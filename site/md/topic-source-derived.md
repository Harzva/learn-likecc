## 各节摘要（对照 HTML）

### 材料边界（必读）

声明所用源码来源的性质（重建、可能不完整）、**不能当作官方行为保证**，并提示读者以官方文档与许可证为准。

### 六条可迁移的「实现哲学」

从具体实现中抽象出六条设计取向（如安全默认、可观测性等），每条 1～3 句话说明**在代码中的落点类型**，避免空洞口号。

### 模块 ↔ 思想（真实路径速查）

用表格把文件/模块路径映射到设计问题（权限、压缩、工具调度等），方便带着问题**反向搜代码**。

### 硬读导航（带锚点）

总起：说明「硬读」= 带着清单逐文件读，而非通读仓库；下列子锚点分主题展开。

#### 权限硬读 / Compact 硬读 / 其他速览

分别指向页内更长的专项段落；其他速览可覆盖 hooks、MCP 客户端等尚未独立成章的线索。

#### 进阶子页：终端 Agent Runtime

把本页里分散的启动、状态、query loop、工具控制面、扩展隔离和上下文治理收成一条主脊柱，适合作为第二轮阅读。

### 权限硬读 · yoloClassifier + permissionExplainer

聚焦权限策略的分类与面向用户的解释管道，连接 S03 权限课与 D 深挖。

#### 策略如何落进 JSON 设置：AutoModeRules

说明规则结构如何影响自动批准/拒绝逻辑，强调**配置即行为**。

#### 面向 UX 的结构化解释：explain_command

说明如何把内部决策翻译成用户可见理由，减少「黑箱拒绝」体验。

#### 走读清单（权限）

列出建议打开的文件/函数顺序，作为自学者 checklist。

### Compact 硬读 · compactConversation

聚焦上下文压缩实现：与 S05 压缩课对照，展示 token 统计、触发点与 hook 协作。

#### 入口即立边界：token 计数与参数面

说明压缩入口如何读取模型窗口与业务参数，错误配置时的典型症状。

#### PreCompact Hook 与指令合并

说明扩展点如何在压缩前后注入策略或用户指令，体现**可插拔设计**。

#### 走读清单（Compact）

给出压缩链路的阅读顺序，与权限走读清单格式一致。
