## 一句话

历史无限增长会顶满上下文；OpenHarness 分层做 **microcompact（便宜清屏）** 与 **LLM 摘要**，并在每轮 `run_query` 开头 `auto_compact_if_needed`；系统提示里再叠 CLAUDE.md / memory / skills，与压缩生命周期不同。

## 关键路径

- `services/compact/__init__.py`：`auto_compact_if_needed`、`COMPACTABLE_TOOLS`、token 阈值。  
- `engine/query.py`：`while` 循环开头调用压缩（接 [OH01](oh01.md)）。  
- `prompts/context.py`：`build_runtime_system_prompt`。

## 源码导读

- 模块注释与 Claude Code microCompact / autoCompact 思路对齐：先清旧 `tool_result` 文本，不够再摘要。  
- 压缩改 `messages` 本体；`build_runtime_system_prompt` 在系统侧叠加项目说明，二者都耗 token 但更新频率不同。
