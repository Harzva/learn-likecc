为什么要把它升成专题

以前“Claude Code 解构”更像一个单页栏目，但现在我们已经不只是在拆 Claude Code 的内部结构，还开始拆 Superset / 外层调度壳层 / meta-agent 这些更高一层的东西。

所以“庖丁解牛”不该再只是 Claude Code 的别名，而应该升级成一个更宽的专题名。它关注的是：怎么把一个复杂系统拆开，看出它的层次、边界和真正的控制点。

推荐阅读顺序

1. 先看 CC

先建立 Claude Code 自己的结构心智：Loop、目录、工具、命令、实验开关。

2. 再看 Superset 解构

再去看外层工作台怎样调 inner agent，这样更容易分清“主脑”和“worker”。

3. 回到 Like Code

最后回到我们的工程，判断哪些该学，哪些不该照搬，哪些要做成 Claude Code 中心化能力。
