像 Hermes Agent、OfficeCLI 这种项目，优先引用官网、官方 GitHub、正式文档与内置技能说明，再补社区解读；不要只靠二手快讯判断其能力边界。

推荐路径

  1. 优先运行 python3 tools/fetch_hot_topic_sources.py --write 刷新热点快照。
  2. 先挑一条真的值得继续讲的条目，再像上面的 Build Agents That Don’t Fail in Production 一样路由进下游热点页，而不是把 snapshot 原样堆在 intake 层。
  3. 按 hot-topic-curation 的路由规则分到 Agent、RAG 或其他专题热点页。
  4. 用模型生成短中文摘要，人工校对后插入表格。
  5. 避免整篇转载；保留原文链接与作者站点署名。
