GitHub 成就 / 图标专题
把开发者身份展示这条小线收起来

这页不是教你“凑 badge”，而是帮我们理解：GitHub 成就为什么会变成开发者文化的一部分，它和工具链、个人主页、profile 展示之间是什么关系，以及社区整理页和官方能力边界在哪里。

为什么站内值得有这条小专题

第一：它是开发者工具链文化的一部分，不只是“好看图标”。

第二：它和 GitHub profile、项目展示、个人品牌表达有关系，适合挂在工具链专题下面。

第三：我们之前其实只留下了样式壳，没有真正把这条线做成页面和入口。

边界说明：官方成就 vs 社区整理页

层是什么当前作用
GitHub 官方 achievementsGitHub 产品里真实存在的成就体系站内讨论的核心对象
社区整理页例如 github-achievements 中文页方便查图标、名称、中文解释，但不是官方产品页
站内专题我们的整理与解释层帮助把 achievements 放回“开发者工具链 / 身份展示”语境里看

也就是说，这条线之前为什么“没展示在网页上”，不是因为有页面没挂出来，而是根本没有正式页面、导航入口和长循环计划，只有一段预留的 badge CSS。
