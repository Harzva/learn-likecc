Thesis

这间铺子怎么和导航站拉开差异

01

按场景分货

不是按“这个站把谁归到哪一类”，而是按我们真实会遇到的问题来收：终端开发、IDE 编排、Agent 框架、API 路由、基准评测。

02

只收会反复打开的入口

不是为了凑数量。每个货架先放一批高复访入口，后续再慢慢扩，而不是一上来铺 100 个名字让人眼花。

03

站内专题互相串门

这里和 Agent 专题、工具链专题、大模型专题互补。杂货铺负责入口与分拣，深挖交给各条专题。

Shelf 01

CLI Agent 货架

这条货架现在不再只按“是不是终端工具”来收。站内已经把它继续拆成两层：先分 workflow-complete shell、TUI-first open shell、artifact-control CLI，再分 action approval、artifact review、dangerous-command gate 这几种 approval surface。

Claude Code

终端里的主流 agentic coding 壳层之一，适合和本站课程主线对照着看。

 Codex

OpenAI 的编程代理产品线入口，CLI / IDE / Web 一体化方向很值得长期观察。

 OpenCode

开源 CLI coding agent，适合拿来观察官方壳层之外的替代路径。

 iFlow CLI

国内终端 Agent 方向的一个现成入口，后面可和其他 CLI 工具做对照。
Qwen CLI

已经进入补货名单，等待核对更合适的官方入口后再挂正式链接。

MiniMax CLI

同样先占位。我们更想收稳定官方入口，而不是临时二手链接。

Agent 开发工具货架

OpenAI Agents SDK

Shelf 05

模型评测货架

MMLU

大模型通用能力老牌基准，哪怕不完美，也还是很多讨论的共同起点。

 MMBench

多模态评测入口之一，你已经点名要收，这里先放成常驻货架。

 OpenCompass

国内看榜和看评测体系都很重要的入口，适合和 MMBench 一起看。

 LMSYS Arena

主观偏好与竞技场榜单入口，适合作为“用户感知”类评测补充。

 HELM

更强调评测框架与覆盖面的代表，适合和单分数榜单区分开看。

 CMMLU

中文评测维度的常见入口，后续可并入“中文能力基准”子货架。
