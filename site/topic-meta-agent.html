Meta-Agent
学会去 agent 一个 agent

今天是 2026-04-09，我们先把这个能力正式命名为 meta-agent。
它关心的不是“多开几个模型”，而是：外层 agent 如何理解任务、拆任务、调度 inner agent、再回收结果。

它到底是什么意思

这里的 `meta-agent` 不是更强一点的模型名字，而是一个结构概念：

外层 agent

负责理解任务、拆任务、决定谁去做、什么时候回收结果。

内层 agent

负责执行具体任务，可以是 Claude Code、Codex、Gemini CLI 等 worker。

关键能力

真正难的不是多开窗口，而是让外层系统持续看见、控制、回收这些 inner agents。

Superset 是怎么做到的

从参考工程看，`Superset` 本质上不是单个 agent，而是一个 agent orchestration shell。它把这五层拼在一起：

1. Workspace / Worktree

给每个任务一个独立容器，解决隔离问题。

2. Terminal

把 CLI agent 变成可拉起、可附着、可回放的运行体。

3. Chat

不是普通聊天框，而是控制协议入口。

4. Panes / Tabs

把 worker 变成可切换、可观察的工作位。

5. Host-Service

真正的外层调度中枢，统一 runtime、事件、terminal、git、fs 状态。

所以 `Superset` 不是“多终端工具”，而是“系统替你管理多个 inner agents”。这就是它的 meta-agent 味道。

Like Code 应该怎么学它

我们不应该照着 `Superset` 再造一个外部总控台。真正要拿走的是：

worker runtime

把外部 CLI agent 抽象成可调用 worker，而不是一堆散乱终端。

统一状态流

pane、subagent、repo/worktree、prompt、progress、result 都要能串起来。

Claude Code 为中心

不是外面再造一个中心，而是让 Claude Code / Like Code 自己学会调度别的 agent。

把什么做到基线，才算产品

这条线上，产品基线不该是“支持很多 agent”这种口号，而应该是这几件事：

可拉起

在主线程里一键拉起外部 worker。

可看见

知道它在哪个 pane、哪个 repo/worktree、当前状态如何。

可控制

支持中断、重试、继续、回收结果。

可回灌

结果能重新并回主线程 transcript，而不是散落在外部终端。

一句话收尾：Superset 教我们“如何在外层调度 agent”；Like Code 要做的是“如何让 Claude Code 自己学会调度 agent”。
