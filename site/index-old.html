agent_loop.ts 
async function agentLoop() {
  while (!finished) {
    // 1. 获取用户输入
    const userMessage = await getUserInput()

    // 2. 调用 Claude API
    const response = await callClaudeAPI(userMessage)

    // 3. 处理工具调用
    if (response.toolCalls) {
      for (const toolCall of response.toolCalls) {
        const result = await executeTool(toolCall)
        context.push(result)
      }
    }

    // 4. 输出响应
    displayResponse(response)
  }
}

💼 面试问题专栏

高频面试题，掌握 Claude Code 核心概念

Q1 概念 简单

什么是 Claude Code？它与 ChatGPT 有什么区别？

核心区别：

  • Claude Code 是 CLI 工具，直接操作本地文件和终端
  • ChatGPT 是 Web 应用，需要手动复制代码
  • Claude Code 具备工具调用能力（读写文件、执行命令）
  • Claude Code 支持持久化记忆和项目级规则
Q2 概念 中等

解释 Claude Code 的 Agent Loop 工作原理

四步循环：

while (!finished) {
  1. getUserInput()    // 获取用户输入
  2. callClaudeAPI()   // 调用 API
  3. executeTools()    // 执行工具
  4. displayResponse() // 输出响应
}

每个循环中，Claude 决定是否需要调用工具，并处理返回结果。

Q3 概念 中等

什么是 MCP (Model Context Protocol)？

MCP 是一种协议，让 Claude Code 连接外部工具和数据源：

  • 定义：标准化的工具集成协议
  • 作用：连接 GitHub、数据库、文件系统等
  • 配置：通过 .mcp.json 文件
  • 管理：/mcp 命令查看和管理

Q6 实践 困难

如何配置 GitHub MCP 集成？

步骤：

# 1. 创建 GitHub Token
# 访问 github.com/settings/tokens
# 创建 token，勾选 repo 权限

# 2. 设置环境变量
export GITHUB_TOKEN="ghp_xxxx"

# 3. 创建 MCP 配置
cat > .mcp.json << 'EOF'
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "$GITHUB_TOKEN"
      }
    }
  }
}
EOF

# 4. 在 Claude Code 中验证
# /mcp

Q10 场景 困难

如何在 CI/CD 中集成 Claude Code？

CI/CD 配置示例：

# .github/workflows/claude-ci.yml
name: Claude CI
on: [push]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run Claude Code
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p "Review the code changes" \
            --permission-mode dontAsk \
            --output review.md

      - name: Upload Review
        uses: actions/upload-artifact@v4
        with:
          name: code-review
          path: review.md

什么是 Source Map？

Source Map 是一张对照表，告诉调试器压缩代码与源码的对应关系。

  • 开发时：源码清晰可读
  • 发布时：代码被压缩、混淆
  • Source Map：建立两者对应关系
