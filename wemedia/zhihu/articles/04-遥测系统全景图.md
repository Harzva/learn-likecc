## 一个令人不安的发现

在分析 Claude Code 泄露的源码时，我们发现了一个庞大的遥测系统。

这不是简单的"使用统计"，而是一个**全方位的数据采集系统**。

## 遥测数据全景图

### 第一类：设备信息

```typescript
interface DeviceTelemetry {
  // 硬件信息
  cpuCores: number,
  totalMemory: number,
  gpuInfo: string,

  // 系统信息
  osType: 'darwin' | 'linux' | 'windows',
  osVersion: string,
  hostname: string,

  // 网络信息
  ip: string,
  timezone: string,
  locale: string,
  language: string,
}
```

**问题**：这些信息可以唯一标识一台设备，即使更换 IP 也能被追踪。

### 第二类：开发环境

```typescript
interface DevTelemetry {
  // Git 配置
  gitUserName: string,
  gitUserEmail: string,
  gitRemoteUrls: string[],

  // 编辑器
  editor: 'vscode' | 'cursor' | 'other',
  editorVersion: string,

  // 项目信息
  projectPath: string,
  projectType: string,
  dependencies: string[],
}
```

**问题**：Git 配置可能暴露你的 GitHub 身份，项目路径可能暴露工作单位。

### 第三类：使用模式

```typescript
interface UsageTelemetry {

```

### 第四类：对话内容

这是最令人担忧的部分：


## 风险评分系统

源码中最关键的部分是风险评分：

```typescript
function calculateRiskScore(events: TelemetryEvent[]): number {
  let score = 0;

  // IP 风险
  if (isVPN(events.ip)) score += 30;
  if (isDataCenterIP(events.ip)) score += 20;
  if (isBlacklistedIP(events.ip)) score += 50;

  // 环境风险
  if (events.locale === 'zh-CN') score += 15;
  if (events.timezone.includes('Asia/Shanghai')) score += 10;

  // 使用模式风险
  if (events.continueCount > threshold) score += 10;
  if (events.negativeWords.length > 0) score += 5;

  // 第三方工具风险
  if (isThirdPartyClient(events.userAgent)) score += 40;

  return Math.min(score, 100);
}
```

### 风险阈值

| 分数范围 | 处理方式 |
|----------|----------|
| 0-30 | 正常服务 |
| 31-50 | 加强监控 |
| 51-70 | 限流处理 |
| 71-100 | 封号处理 |

## 如何保护隐私？

### 方法一：禁用遥测

源码中有一个开关：

```typescript
const TELEMETRY_DISABLED = process.env.CLAUDE_CODE_DISABLE_TELEMETRY === '1'
```

设置环境变量：

```bash
export CLAUDE_CODE_DISABLE_TELEMETRY=1
```

**但要注意**：这可能被标记为异常行为。

### 方法二：使用开源替代

- **OpenClaw**：完全本地运行，无遥测
- **Codex**：OpenAI 开源，数据不上报

### 方法三：网络隔离

```bash
# 在防火墙中阻止遥测域名
sudo echo "127.0.0.1 stats.anthropic.com" >> /etc/hosts
```
