作为一名内容创作者，我一直希望能有一套自动化的工具，可以帮助我在多个平台同步发布文章。最近，我成功搭建了一套知乎自动发布系统，全程无需API，完全基于浏览器自动化技术。今天把整个实现过程记录下来，分享给有需要的朋友。

一、背景与需求
为什么要做自动发布？

作为同时运营公众号、微博、知乎的创作者，最大的痛点就是内容分发。一篇好不容易写出来的文章，需要手动复制到各个平台，调整格式、上传图片、填写标题...重复性工作占据了大量时间。

我的核心需求：

一键将Markdown文件发布到知乎
2. 支持封面图自动上传
3. 不需要申请API（知乎API门槛较高）
4. 可以集成到自动化工作流
## 二、技术方案选择
调研了几种方案：
方案1：知乎官方API
优点：稳定、正规
缺点：需要申请权限，个人很难通过
方案2：模拟HTTP请求
优点：速度快、资源占用低
缺点：知乎的反爬机制很强，cookie容易失效，维护成本高
方案3：浏览器自动化（最终选择）
优点：模拟真人操作，不容易被封，稳定性好
缺点：需要安装Chrome，占用资源稍高
最终选择了Puppeteer + Puppeteer-Extra-Stealth的方案，这也是目前最成熟的浏览器自动化方案。
## 三、具体实现过程
### 1. 环境准备
首先需要安装依赖：
```
npm install puppeteer-extra puppeteer-extra-plugin-stealth puppeteer-core
```
### 2. Cookie配置（最关键的一步）
知乎的登录验证比较严格，需要手动获取cookie：
- 打开Chrome浏览器，访问知乎创作中心
- 登录账号
- 使用Cookie编辑器插件导出关键cookie
- 主要是z_c0、SESSIONID、d_c0这几个字段
将cookie保存为JSON格式，供后续脚本使用。
### 3. 核心代码编写
整个发布流程分为几个步骤：
第一步：打开知乎编辑器
使用puppeteer.launch启动浏览器，设置headless为false方便调试，加载之前保存的cookie。
第二步：输入标题
知乎使用的是Draft.js富文本编辑器，不能直接操作DOM，必须使用键盘输入才能触发保存。这里踩了一个坑，最初用evaluate直接设置value，结果编辑器不识别，必须改用page.keyboard.type逐字输入。
第三步：输入正文
同样需要键盘输入。为了提高效率，我采用了分段输入的方式，每输入一段按一次回车，同时加上适当的延迟，模拟真人打字节奏。
第四步：点击发布
滚动到页面底部，找到发布按钮并点击。这里需要注意，知乎的发布按钮在点击后会显示"发布中..."，需要等待页面跳转才算真正发布成功。
### 4. 遇到的问题和解决方案
问题1：标题输入失败
最初使用document.querySelector找到输入框后直接用evaluate设置value，发现标题虽然在输入框里显示，但编辑器没有真正保存。
解决：改为使用page.click先点击输入框获取焦点，再用page.keyboard.type键盘输入。
问题2：正文内容显示但字数为0
和标题类似，直接操作DOM的内容不会被Draft编辑器识别。
解决：全部采用键盘输入，逐段输入正文内容，确保触发编辑器的input事件。
问题3：发布按钮找不到
知乎的按钮class名是动态生成的，用固定选择器会失败。
解决：通过按钮文字内容查找，优先找"发布"按钮，找不到就找最后一个可见按钮（通常是发布）。
## 四、使用方法
现在整个工具已经封装成了命令行工具，支持三种使用方式：
方式1：Markdown文件
```
node publish_article.js ./article.md
```
方式2：JSON配置
```
node publish_article.js --config ./config.json
```
方式3：命令行参数
```
node publish_article.js --title "标题" --content "内容" --cover ./cover.jpg
```
## 五、定时任务集成
配合crontab或OpenClaw的定时任务功能，可以实现定时自动发布。比如每天早上8点自动抓取热点，生成文章后自动发布到知乎。
## 六、注意事项
1. Cookie有效期：知乎的cookie会过期，需要定期更新
2. 发布频率：建议每篇文章间隔5分钟以上，避免触发反spam机制
3. 内容审核：知乎有内容审核机制，发布后可能需要等待审核
4. 调试技巧：发布失败时会在目录生成debug截图，方便排查问题
## 七、总结
通过浏览器自动化技术，我们实现了一套无需API的知乎自动发布系统。整个过程虽然踩了不少坑，但最终效果还是很满意的。现在我可以专注于内容创作，把繁琐的分发工作交给自动化工具。
这套方案不仅适用于知乎，稍作修改也可以用于其他平台，比如小红书、公众号等。核心思路都是一样的：模拟浏览器操作，绕过API限制。
如果你也想搭建类似的自动化工具，欢迎在评论区交流讨论！
